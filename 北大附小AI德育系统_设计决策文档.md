## 一、项目背景

### 目标
- **0518** 三年级一个班级落地试用，核心功能跑通可用
- **0528** 校长论坛展示，陈述产品架构和AI赋能点，核心功能视频/实地展示
- **后续** 进行2-3个版本迭代，最终打造有北大附小特色的AI德育系统

### 产品定位
这是**产品化探索**，不是单纯的项目交付。北大附小是第一个标杆客户，系统架构需要考虑未来复制到其他学校的可能性。

### 决策 #4：交作业流程

| 问题 | 大屏"交作业/划号单"功能，文档提到"老师发布作业，学生点击交作业，姓名消失"。这个流程怎么工作？ |
|------|------|
| **选项A** | 学生主动操作 — 学生走到大屏前点击自己名字来交作业 |
| **选项B** | 老师代操作 — 老师在平板端批量勾选已交作业的学生，同步到大屏 |
| **选项C** | 两种都支持 — 学生可以自己点，老师也可以批量勾选 |
| **决策** | **选项C：两种都支持** |
| **理由** | 更灵活，适应不同教学场景。学生自主交作业培养自律性，老师批量操作提高效率 |

### 决策 #5：AI电话亭入口

| 问题 | "AI电话亭"是学生心理倾诉功能。文档提到"老师选择学生姓名进入"。这个功能放在哪个端？ |
|------|------|
| **选项A** | 平板端（老师版） — 老师平板上有1个入口，选择学生后学生开始对话 |
| **选项B** | 大屏端 — 在大屏上有"我想说"入口，学生点击后进入对话 |
| **选项C** | 两者都有 — 大屏有入口，平板也可以让学生使用 |
| **决策** | **选项A：平板端（老师版）** |
| **理由** | 更可控，老师可以监督学生使用，避免大屏端隐私问题。学生在老师引导下使用平板进行AI对话 |

### 决策 #8：大屏与平板联动方式

| 问题 | 老师在平板端进行点评后，大屏应该如何响应？ |
|------|------|
| **选项A** | 实时同步+动效 — 老师提交点评后，大屏实时显示"+2分"动效，学生能立即看到 |
| **选项B** | 定时刷新 — 大屏每30秒刷新一次数据，不用实时推送 |
| **选项C** | 手动刷新 — 大屏有刷新按钮，老师/班长点击后更新数据 |
| **决策** | **选项A：实时同步+动效** |
| **理由** | 即时反馈是学生激励的核心。老师点评后学生能立刻看到加分动效，形成正向激励闭环 |

### 决策 #9：平板端登录方式

| 问题 | 平板端是多个老师共用的，需要支持不同学科老师（语文、数学等）切换账号，以便记录"是谁"进行的点评。登录方式怎么设计？ |
|------|------|
| **决策** | **手机号登录 + 多账号快速切换** |
| **具体方案** | |

**登录流程：**
1. **主登录方式**：手机号验证码 / 手机号+密码
2. **多账号切换**：本地存储多个已登录账号信息，支持快速切换（类似微信切换账号）
3. **退出机制**：支持退出当前账号，重新登录其他账号

**设计考量：**
- 系统可以记录不同学科老师对学生的评价明细，便于后续分析
- 快速切换减少操作摩擦，提高日常使用效率

---

### 决策 #10：大屏端登录方式

| 问题 | 大屏固定在教室，每天都要用。但无法获取电视设备码进行绑定，且班级可能因升年级更换教室。怎么设计？ |
|------|------|
| **决策** | **班主任登录 + 记住账号和班级** |
| **具体方案** | |

**登录流程：**
1. **首次使用**：班主任扫码 或 手机号登录，选择班级
2. **记住状态**：默认记住账号和所选班级，不需要每天登录
3. **切换支持**：可以切换班级 或 退出重新登录其他账号
4. **适应升级**：班级换教室时只需重新选择班级即可

**设计考量：**
- 大屏呈现的是班级综合的点评结果，以班主任身份统领整个班级多学科数据
- 避免每天使用都需要重复登录操作

## 五、系统架构约束

### 产品化考量
- 德育指标体系：**可配置**（不硬编码北大附小的四维度）
- 品牌色：**可配置**（支持不同学校的品牌色定制）
- 模块命名：**可配置**（支持不同学校的命名偏好）

### 技术约束
- 大屏端与平板端：**实时同步**（WebSocket或类似技术）
- AI对话内容：需要**敏感词识别**和**预警触发机制**
- 数据安全：沟通记录提到"双人审核机制"和"数据隔离"（待进一步明确）
