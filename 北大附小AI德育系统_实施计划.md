# 北大附小AI德育系统 — 高保真页面设计实施计划

## Context

北大附小AI德育系统需要在0518上线三年级一个班级试用，0528校长论坛展示。本计划基于前期需求讨论和11项关键设计决策，输出三端（大屏端、平板端、网页端）的高保真静态页面，用于产品需求陈述和团队评审。

**设计约束：**
- 品牌主色：深红色（参考系统首页参考.png）
- 大屏端：避免大色块红色，使用马卡龙色系作为辅助色
- 目标用户：小学生（大屏端），需要活泼、易识别的视觉风格
- 技术要求：自包含HTML文件，可直接在浏览器中查看
- **平板端双端兼容**：需要同时适配手机App(375px)和平板App(768px+)

## 设计规范

### 色彩系统

```
品牌主色（深红）: #8B0000 / #A52A2A
辅助色（马卡龙色系）:
  - 薄荷绿: #98D8C8
  - 柠檬黄: #F7DC6F
  - 樱花粉: #FFB6C1
  - 天空蓝: #87CEEB
  - 薰衣草紫: #E6E6FA

功能色:
  - 成功/加分: #4CAF50
  - 警告: #FF9800
  - 错误/扣分: #F44336
  - 信息: #2196F3

中性色:
  - 背景: #F5F5F5
  - 卡片: #FFFFFF
  - 文字主色: #333333
  - 文字次色: #666666
```

### 大屏端设计要点

1. **字体大**：标题 48-72px，正文 24-32px，确保远距离可读
2. **高对比度**：深色文字配浅色背景
3. **圆角卡片**：圆润的视觉风格，适合小学生
4. **动效提示**：加分时显示 "+2分" 飘动动画
5. **分组展示**：按小组展示学生，每组不同颜色标识

### 平板端设计要点

1. **触控友好**：按钮最小 44x44px
2. **清晰层级**：信息层级分明，减少认知负担
3. **快捷操作**：常用功能一键可达
4. **响应式布局**：
   - 手机端(375px)：单列布局，底部Tab导航
   - 平板端(768px+)：多列布局，侧边栏导航
5. **与大屏端复用**：行为点评页与大屏端点评模块使用相同布局和交互逻辑

### 网页端设计要点

1. **专业简洁**：管理后台风格
2. **数据可视化**：图表展示统计数据
3. **表格为主**：列表和表格展示管理数据

## 验证方式

1. **视觉验证**：在浏览器中打开每个HTML文件，检查布局和样式
2. **响应式检查**：
   - 大屏端：1920x1080 分辨率
   - 平板端-平板视图：1024x768 或 iPad 尺寸
   - 平板端-手机视图：375x812（iPhone）或 360x640（Android）
   - 网页端：1440x900 桌面分辨率
3. **参考图对比**：与Excel中的功能参考图进行对比，确保关键元素一致
4. **决策一致性**：检查页面是否符合11项设计决策
5. **大屏/平板点评一致性**：确保行为点评页与大屏端点评模块布局一致
