# 北大附小AI德育系统 - 大屏端 PRD

## 一、需求背景与关键决策

### 1.1 项目背景

**里程碑目标**：
- **0518** 三年级一个班级落地试用，核心功能跑通可用
- **0528** 校长论坛展示，陈述产品架构和AI赋能点
- **后续** 进行2-3个版本迭代，打造有北大附小特色的AI德育系统

**产品定位**：这是**产品化探索**，不是单纯的项目交付。北大附小是第一个标杆客户，系统架构需要考虑未来复制到其他学校的可能性。

### 1.2 关键设计决策

#### 决策 #1：大屏激励形式

| 问题 | 大屏端的"学生激励"形式用哪种方案？ |
|------|------|
| **决策** | **分数/排行榜为主** — 简化开发，先用数字+动效，激励元素二期迭代 |
| **理由** | 0518时间紧迫，先确保核心功能可用，种树养花作为二期亮点功能 |

#### 决策 #4：交作业流程

| 问题 | 大屏"交作业/划号单"功能的操作流程？ |
|------|------|
| **决策** | **两种都支持** — 学生可以自己点，老师也可以批量勾选 |
| **理由** | 更灵活，适应不同教学场景。学生自主交作业培养自律性，老师批量操作提高效率 |

#### 决策 #5：AI电话亭入口

| 问题 | "AI电话亭"学生心理倾诉功能放在哪个端？ |
|------|------|
| **决策** | **平板端（老师版）** — 老师平板上有入口，选择学生后开始对话 |
| **理由** | 更可控，老师可以监督学生使用，避免大屏端隐私问题 |

#### 决策 #7：德育指标体系可复制性

| 问题 | 如果未来推广到其他学校，德育维度应该怎么处理？ |
|------|------|
| **决策** | **可配置** — 德育指标体系在系统架构层面设计成可配置的 |
| **理由** | 产品化探索需要考虑可复制性。北大附小的四维度（快乐/进取/儒雅/大气）是第一个配置示例 |

#### 决策 #8：大屏与平板联动方式

| 问题 | 老师在平板端进行点评后，大屏应该如何响应？ |
|------|------|
| **决策** | **实时同步+动效** — 老师提交点评后，大屏实时显示"+2分"动效 |
| **理由** | 即时反馈是学生激励的核心。老师点评后学生能立刻看到加分动效，形成正向激励闭环 |

#### 决策 #10：大屏端登录方式

| 问题 | 大屏固定在教室，每天都要用，怎么设计登录？ |
|------|------|
| **决策** | **班主任登录 + 记住账号和班级** |
| **具体方案** | 首次使用班主任扫码/手机号登录，选择班级；默认记住状态，不需要每天登录；可切换班级或退出重新登录 |

## 二、产品概述

### 2.1 产品定位
教室大屏展示端，面向班级学生和老师，实时展示德育积分数据，形成"行为记录→积分激励→榜单展示"的正向反馈闭环。

### 2.2 核心价值
- **即时激励**：学生行为得分后大屏实时展示动效，强化正向行为
- **公开透明**：班级积分数据可视化，营造良性竞争氛围
- **操作便捷**：老师可直接在大屏完成点评、作业提交等日常操作

---

## 三、用户角色

| 角色 | 场景 | 核心诉求 |
|------|------|----------|
| 班主任 | 课堂点评、检查作业 | 快速完成学生评价，激励学生 |
| 学生 | 课间查看排行榜 | 了解自己和小组的积分排名 |

> **说明**：科任老师只在平板端进行学科点评，不在大屏端操作。

## 四、功能模块

### 4.1 登录页

**用户场景**：班主任在教室大屏上登录系统

**核心流程**：
1. 手机号+验证码登录
2. 登录成功后跳转首页
3. 系统自动记住登录状态（localStorage）

**关键规则**：
- 登录后获取班主任所带班级列表
- 默认选中第一个班级

> **备注**：微信扫码登录作为二期功能规划。

---

### 4.2 首页

**用户场景**：班主任登录后的功能入口页

**核心功能**：
| 入口 | 说明 |
|------|------|
| 进入点评 | 跳转点评页，开始学生评价 |
| 排行榜 | 查看班级积分排行 |
| 交作业 | 记录学生作业提交情况 |
| 我想说 | 提示"请前往老师平板端使用" |

**页面元素**：
- 左上角：校徽 + 学校名称（"北京大学附属小学"）
- 右上角：班级切换选择器 + 班主任头像（显示姓氏）

> **说明**：首页使用独立布局（渐变背景、大卡片入口），不复用通用顶部导航。

---

### 4.3 点评页

**用户场景**：班主任在课堂上对学生进行即时点评

**核心流程**：
1. 选择查看维度：小组 / 全部学生 / 座位表
2. 点击学生卡片选中（支持多选）
3. 选择点评指标（加分/减分项）
4. 确认点评，触发加分动效

**功能详情**：

| 功能点 | 说明 |
|--------|------|
| 小组视图 | 按小组展示学生，每组显示组名、组图标、组总分 |
| 全部学生 | 网格展示全班学生 |
| 座位表 | 按教室座位布局展示（布局由平板端配置） |
| 学生卡片 | 显示：姓名、当前积分；点击可选中 |
| 小组编辑 | 可修改小组名称、选择小组图标（20个候选图标） |
| 加分动效 | 点评成功后，被加分学生卡片显示"+N"红色飘动动画（#EE595B渐变） |
| 时间筛选 | 今日/本周/本月/本学期 |

**点评指标体系**：
- 四个维度Tab切换：快乐（身心健康）/ 进取（创新素养）/ 儒雅（儒雅品格）/ 大气（责任担当）
- 每个维度包含加分项和扣分项，来源于《行为指标规则》
- 指标以两列区块按钮布局展示，先加分后扣分
- 班主任自定义创建的"特色指标"（`custom: true`）优先展示，右上角标注"特色"标签

**交互规则**：
- 加分：红色主题正向动效（+N飘动，卡片脉冲高亮）
- 减分：低调处理，仅数字变化，无负面动效

---

### 4.4 排行榜页

**用户场景**：展示班级德育积分排名，激励学生

**核心流程**：
1. 选择榜单维度：个人榜 / 小组榜 / 单项榜
2. 选择统计周期：今日 / 本周 / 本月 / 本学期
3. 查看排名详情

**功能详情**：

| 榜单类型 | 展示内容 |
|----------|----------|
| 个人榜 | 全班学生积分排名，前三名领奖台（金银铜皇冠+圆形徽章），其余学生3列列表布局，奇偶行交替背景色 |
| 小组榜 | 各小组3列卡片排名，每组展示组员及各自得分（2列内嵌网格） |
| 单项榜 | 5个维度（作业 / 快乐 / 进取 / 儒雅 / 大气）分别展示TOP5，3列卡片布局 |

**页面布局**：
- 左侧：装饰图区域（480px，领奖台图片垂直居中）
- 右侧：排行榜内容区（白色半透明背景面板 `rgba(255,255,255,0.65)`，圆角24px）

**视觉设计**：
- 温暖橙红色系配色（`#FDF2F3` 页面底色）
- 排名列表奇数行背景 `rgba(#F94348, 0.1)`，偶数行透明
- 圆润卡片样式，悬停上浮效果
- 金币图标（SVG）跟随分数展示

---

### 4.5 交作业页

**用户场景**：班主任记录学生作业提交情况

**核心流程**：
1. 左侧选择/创建作业
2. 右侧按小组展示学生名单
3. 点击学生标记为已提交，触发加分

**功能详情**：

| 功能点 | 说明 |
|--------|------|
| 作业列表 | 左侧列表（320px）展示今日作业，点击切换 |
| 创建作业 | 弹窗面板：选择学科（语文/数学/英语/其他，区块按钮选择）、输入标题、设置截止时间（默认当天17:00本地时间） |
| 学生名单 | 右侧按小组分组展示，3列 groups-grid，每组内4列 students-grid |
| 提交标记 | 点击学生 → 显示加分动效 → 状态变为已提交 |
| 提交进度 | 顶部信息栏显示已交/总数 + 进度条，以及当前加分规则状态（按时/超时） |

**加分规则**（自动根据截止时间判断）：
| 提交情况 | 积分 | 大屏展示 |
|----------|------|----------|
| 按时提交（当前时间 ≤ 截止时间） | +2分 | 绿色标签"按时提交+2分"，动效显示"+2分" |
| 超时提交（当前时间 > 截止时间） | +1分 | 橙色标签"已超时 · 提交+1分"，动效显示"+1分" |
| 未提交 | -1分 | 后台处理，不在大屏展示负面效果 |

**三种交互模式**（供客户演示选择）：
1. **灰显保留模式**（张书斌组演示）：点击后显示加分红色飘动动效 → 卡片灰显，保留姓名，右下角显示红色半透明"✓"
2. **语文作业模式**：点击后显示"✓"弹出动画 → 姓名消失但卡片位置保留（空白占位）
3. **标准模式**（数学/英语/其他）：点击后显示加分红色飘动动效 → 卡片变为透明（位置保留）

## 七、版本迭代规划

### 7.1 一期功能（0518）
- 系统展示页（默认首页 `/`，品牌展示入口）
- 登录页（手机号+验证码）
- 首页（功能入口卡片）
- 点评页（小组视图、全部学生、座位表；四维度行为指标体系；特色指标优先展示）
- 排行榜页（个人榜、小组榜、单项榜；左侧领奖台装饰+右侧白色面板）
- 交作业页（三种提交交互模式；按时/超时差异化加分）

### 7.2 二期功能

**数据维度升级**：学生数据增加**积分+经验**双重维度

| 维度 | 获取方式 | 消费方式 |
|------|----------|----------|
| 积分 | 点评、交作业行为触发 | 积分兑换（兑换实物/虚拟奖励） |
| 经验 | 点评、交作业行为触发 | 经验养树（浇灌成长树） |

**新增功能模块**：
1. **积分兑换**：学生用积分兑换奖励，形成激励闭环
2. **经验养树**：环保主题成长激励，符合校方要求的种树/养花形式

### 7.3 后续迭代
1. **实时推送**：WebSocket接入，平板端点评后大屏实时刷新
2. **数据看板**：班级概况统计图表（饼图、趋势图）
